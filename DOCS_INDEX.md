## üéØ Task-Based Navigation

### I want to...

**...set up my portfolio**
1. Clone the repo
2. Follow [GITHUB_PAGES_SETUP.md](GITHUB_PAGES_SETUP.md)
3. Use [CHECKLIST.md](CHECKLIST.md) to verify

**...add a blog post**
1. Read "Adding a Blog Post" in [QUICK_REFERENCE.md](QUICK_REFERENCE.md)
2. Create file in `_posts/YYYY-MM-DD-slug.md`
3. Copy front matter from existing post
4. Write your content
5. Push to GitHub

**...customize the design**
1. Check [QUICK_REFERENCE.md](QUICK_REFERENCE.md) "Customization Quick Tips"
2. Edit `assets/css/style.css`
3. Test locally: `bundle exec jekyll serve`
4. Push changes

**...add a project**
1. Read "Adding a Project" in [QUICK_REFERENCE.md](QUICK_REFERENCE.md)
2. Create file in `_projects/NN-slug.md`
3. Follow project front matter format
4. Write project description
5. Push to GitHub

**...understand the site structure**
1. Review [SITEMAP.md](SITEMAP.md)
2. Look at [ARCHITECTURE.md](ARCHITECTURE.md)
3. Explore files in repository

**...debug a problem**
1. Check troubleshooting in [CHECKLIST.md](CHECKLIST.md)
2. Review "Troubleshooting" in [GITHUB_PAGES_SETUP.md](GITHUB_PAGES_SETUP.md)
3. Check [README.md](README.md) "Troubleshooting" section

**...test locally**
1. See "Local Development" in [README.md](README.md)
2. Use `bundle exec jekyll serve`
3. Visit `http://localhost:4000`
